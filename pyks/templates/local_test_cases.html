This page presents examples of possible query URLs defined by the The OpenPGP HTTP Keyserver Protocol (HKP) which is currently (2014) available as an Internet-Draft from David Shaw from March 2003 and is implemented bei OpenPGP implementation such as GnuPG or PGP. These query URLs are also used by the keyserver features of your OpenPGP implementation, e.g.

  • Mozilla Thunderbird with the Enigmail plugin
  • GnuPG command line tool
  • Android App APG - Android Privacy Guard
  • etc...
Therefore you should point your OpenPGP client to this keyserver (probably running at http://localhost:11371. Unfortunally not all of the currently existing OpenPGP clients allow you to configure your own keyserver. Instead they have a built-in list of predefined keyservers, such as
  • pool.sks-keyservers.net
  • subkeys.pgp.net
  • pgp.mit.edu
  • etc...

Submitting Keys To A Keyserver

See HKP Specification Draft how for this.

/pks/add

Corresponding GnuPG commands

gpg --keyserver hkp://localhost:11371 --send-keys 0xFBB75451

Corresponding Thunderbird/Enigmail use cases

Using Thunderbird/Enigmail to upload keys to key server results requests of the following pattern: 

/pks/add HTTP/1.1

Example Key 1 - Max.Mustermann@localhost.de

keytext:

>

Example Key 2 - Ubuntu Signing Key

keytext:

Requesting Data From A Keyserver

See HKP Specification Draft how for this.

index-Operation

See HKP Specification Draft how for an explaination or this example GET-query to retrieve the Upuntu signing key:

  • /pks/lookup op=index options=mr exact=off search=0xFBB75451
  • /pks/lookup op=index options=mr exact=on search=cdimage@ubuntu.com
  • /pks/lookup op=index options=mr exact=on search=C5986B4F1257FFA86632CBA746181433FBB75451
... or the other test key:
  • /pks/lookup op=index options=mr exact=off search=0x448701A8
  • /pks/lookup op=index options=mr exact=on search=max.mustermann@localhost.de
  • /pks/lookup op=index options=mr exact=on search=5E8C8A788F2685BB19306180890A4614448701A8
Alternative all stored keys can be listed via:
  • /pks/lookup op=index options=mr exact=off
  • /pks/lookup op=index options=mr exact=on
  • /pks/lookup op=index exact=off (HTML-View)

SKS keyservers
  • HKPS /pks/lookup op=index search=0x5B964AE610D49726 options=mr
  • HKPS /pks/lookup op=index search=0x5B964AE610D49726
  • HKP /pks/lookup op=index search=0x5B964AE610D49726 options=mr
  • HKP /pks/lookup op=index search=0x5B964AE610D49726

Corresponding GnuPG commands

These GnuPG commands use the index-operation: 

gpg --keyserver hkp://localhost:11371 --search-keys 0xFBB75451
gpg --keyserver hkp://localhost:11371 --search-keys 0x4E4D8ABFD1AD524D
gpg --keyserver hkp://localhost:11371 --search-keys willi

Corresponding Thunderbird/Enigmail use cases

Using Thunderbird/Enigmail to search keys for all contacts from key server results in dozen requests of the following pattern: 

/pks/lookup?op=index&options=mr&search=some.name%40example.org HTTP/1.1

get-Operation

See HKP Specification Draft how for this.
  • /pks/lookup op=get options=mr exact=off search=0xFBB75451
  • /pks/lookup op=get options=hexview exact=off search=0xFBB75451 (this option is NOT part of HKP!)
SKS keyservers
  • HKPS /pks/lookup op=get search=0x5B964AE610D49726 options=mr
  • HKPS /pks/lookup op=get search=0x5B964AE610D49726
  • HKP /pks/lookup op=get search=0x5B964AE610D49726 options=mr
  • HKP /pks/lookup op=get search=0x5B964AE610D49726

Corresponding GnuPG commands

These GnuPG commands use the get-operation: 

gpg --keyserver hkp://localhost:11371 --recv-keys 0xFBB75451

Corresponding Thunderbird/Enigmail use cases

Using Thunderbird/Enigmail to refresh keys from key server results in the following request: 

/pks/lookup?op=get&options=mr&search=0x4E4D8ABFD1AD524D HTTP/1.1

vindex-Operation

See HKP Specification Draft how for this. Local tests:
  • /pks/lookup op=vindex options=mr exact=off search=0xFBB75451
  • /pks/lookup op=vindex exact=off (HTML-View)
Remote SKS tests
  • /pks/lookup op=vindex search=0x73776167 fingerprint=on
http://127.0.0.1:11371/pks/lookup?op=stats

stats-Operation

The "stats" operation was not specified in HKP Internet Draft by David Shaw. It seems to be a defacto standard followed at least by "SKS" and "Hockeypuck". Local tests:
  • /pks/lookup op=stats
Remote SKS and Hockeypuck tests
  • /pks/lookup op=stats
  • /pks/lookup op=stats
